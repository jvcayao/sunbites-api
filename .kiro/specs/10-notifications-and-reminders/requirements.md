# Spec 10 — Notifications & Payment Reminders

## Overview

A two-sided notification system: staff in the POS app manually send advance payment reminders to parents of subscription students; parents receive and manage those notifications in the parent portal.

The reminder window opens a configurable number of days before a school month starts (default: 14 days). This value is stored in `system_configurations` as `payment_reminder_days`.

---

## Data Model

### `notifications` table (Laravel built-in)

Generated by `php artisan notifications:table`. Stores in-app notifications for `ParentUser`.

```
notifications
  id            (uuid, PK)
  type          (string)     — FQCN of the notification class
  notifiable_type (string)   — "App\Models\ParentUser"
  notifiable_id   (bigint)
  data          (json)       — notification payload
  read_at       (timestamp, nullable)
  created_at, updated_at
```

No soft deletes — individual and bulk deletions use hard delete.

### `parent_payment_reminders` table

Tracks which parents have already been sent a reminder for a given school month + year, to prevent duplicate sends.

```
parent_payment_reminders
  id
  parent_user_id    (FK → parents.id)
  branch_id         (FK → branches.id)
  school_month      (string)     — 'june' | 'july' | ... | 'march'
  school_year       (integer)    — calendar year when the month falls
  sent_at           (timestamp)
  sent_by_user_id   (FK → users.id)
  created_at, updated_at

  UNIQUE (parent_user_id, branch_id, school_month, school_year)
```

## Business Rules

### Reminder Window

- The reminder window for a school month opens `payment_reminder_days` days before the 1st of that month's calendar date.
- Example: August 1 reminder window opens July 18 (if `payment_reminder_days = 14`).
- School months are defined in `config/sunbites.php` `school_months` array (June–March).
- Only subscription students' parents qualify.
- Non-subscription students' parents must never receive these notifications.

### POS Notification Bell

- The POS header contains **exactly one** notification bell (`NotificationBell`). It shows the total unread inbound staff notification count (announcements + pre-registration alerts). Badge is hidden when count is 0. Navigates to `/notifications`.
- The outbound payment-reminder urgency count (parents not yet sent a reminder) lives exclusively in the **Reminders nav item** (sidebar) and the `/reminders` page — NOT in the header bell.
- The bell badge updates in real time via Reverb broadcast events for all inbound staff notification types.

### Sending Reminders

- Admin or Manager selects one or more parents on the Reminders page and clicks "Send Reminder". Supervisors can view the Reminders page but cannot send (the Send button is hidden for their role).
- If a parent in the selection was already sent a reminder for the current month, a warning is shown listing the duplicates. Staff can proceed or cancel.
- On confirmation, a `ParentPaymentReminder` record is created per parent and an in-app notification is written to `notifications` for each selected parent.
- One notification per parent per send action — not one per student. The notification lists all of the parent's subscription students enrolled in the branch with each student's amount from their `StudentMonthlyPayment` record for that month.
- Example content: "Hi Maria, this is a reminder that your child's subscription payment is due before August 1, 2026. Juan Santos — ₱2,430. Please settle at the canteen."
- Amounts come exclusively from `StudentMonthlyPayment.amount` — never recomputed from config — so they reflect any per-branch overrides set at enrollment time.

### Parent Portal Notifications

- Parents see only notifications addressed to them.
- Unread notifications increment the bell badge. The bell subscribes to **all** parent notification types (`PaymentReminderNotification` + `AnnouncementNotification`) in real time via Reverb.
- Opening (clicking) an individual notification marks it as read (`read_at = now()`) optimistically, then persists via API.
- Clicking a `PaymentReminderNotification` card navigates to `/payments`.
- Clicking an `AnnouncementNotification` card expands the full message inline (accordion) and marks it as read.
- Parents can delete individual notifications (hard delete).
- "Mark all as read" sets `read_at = now()` on all unread notifications.
- "Clear all" hard-deletes all notifications for the parent.
- Bell badge count = count of notifications where `read_at IS NULL`.

### Portal Notification Page Design (MagicBell reference)

Each notification card displays:
- Left unread dot (purple, hidden once read)
- Bold type-aware title
- 2-line message preview (type-aware)
- Right-aligned relative timestamp ("just now", "5m", "2h", "3d", "Jun 10")
- `...` context menu per row: "Mark as read", "Delete"
- Header: "Notifications" heading, mark-all-read icon, settings icon
- Empty state when list is empty: bell illustration + "You're all caught up"

Type-aware rendering:

| Notification type | Title | Preview |
|---|---|---|
| `PaymentReminderNotification` | `"Payment Reminder — {Month} {Year}"` | `"{N} student(s) — ₱{total}"` |
| `AnnouncementNotification` | `data.title ?? "Announcement"` | `data.message` (truncated) |

### Payment History (Parent Portal)

- Visible only for subscription students.
- Lists all `StudentMonthlyPayment` records for the student, ordered by year + month order.
- Shows: Month, Amount, Status (paid / unpaid), Paid Date (if paid).
- Accessible from the student detail page in the parent portal.

### Payment History (POS Reminder Detail)

- On the Reminders page, clicking a parent row opens a detail page.
- This page mirrors `/references/parents/{id}` and additionally shows a "Payment History" section per linked subscription student.

## API Routes

### Parent Portal (`auth:parents`)

| Method | Route | Description |
|---|---|---|
| GET | `/api/v1/portal/notifications` | List all notifications (undeleted), newest first |
| GET | `/api/v1/portal/notifications/unread-count` | Returns `{ count: N }` for bell badge |
| PATCH | `/api/v1/portal/notifications/{id}/read` | Mark individual as read |
| DELETE | `/api/v1/portal/notifications/{id}` | Delete individual notification |
| POST | `/api/v1/portal/notifications/mark-all-read` | Mark all as read |
| DELETE | `/api/v1/portal/notifications` | Clear all (hard delete) |
| GET | `/api/v1/portal/students/{student}/payment-history` | Subscription payment history for a student |

### POS / Admin (`auth:sanctum` + ability `staff`)

| Method | Route | Roles | Description |
|---|---|---|---|
| GET | `/api/v1/reminders/bell-count` | all staff | Count of eligible unsent parents in active branch |
| GET | `/api/v1/reminders/eligible-parents` | admin\|manager\|supervisor | Paginated list; branch-scoped |
| POST | `/api/v1/reminders/send` | admin\|manager | Send reminders to selected parent IDs |
| GET | `/api/v1/reminders/parents/{parent}` | admin\|manager\|supervisor | Parent detail + student info + payment history |
