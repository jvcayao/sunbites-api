# Phase 6 ŌĆö Storage (Laravel Cloud Buckets)

Student photos and any uploaded files are stored in **Laravel Cloud Buckets** ŌĆö S3-compatible object storage built into the platform. No separate AWS S3 account needed.

> The default `FILESYSTEM_DISK=local` will NOT work on Laravel Cloud. Container storage is ephemeral ŌĆö files stored locally are wiped on every redeploy. Cloud buckets are the required solution.

---

## 6.1 Install the S3 Adapter

Laravel Cloud Buckets use the S3 driver under the hood. Install the Flysystem adapter:

```bash
composer require league/flysystem-aws-s3-v3
```

Commit the updated `composer.json` and `composer.lock`.

---

## 6.2 Create Buckets

Create a separate bucket for each environment.

### Staging Bucket

```bash
cloud bucket:create --json -n
```

When prompted:
- **Name:** `sunbites-staging`
- **Environment:** `staging`

### Production Bucket

```bash
cloud bucket:create --json -n
```

When prompted:
- **Name:** `sunbites-production`
- **Environment:** `production`

Laravel Cloud automatically injects the following credentials into each environment when a bucket is attached ŌĆö **do not set these manually**:

- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `AWS_DEFAULT_REGION`
- `AWS_BUCKET`
- `AWS_ENDPOINT`
- `AWS_USE_PATH_STYLE_ENDPOINT`

---

## 6.3 Verify Bucket Config in Laravel

Confirm [config/filesystems.php](../config/filesystems.php) has the standard `s3` disk entry (it is included by default in Laravel):

```php
's3' => [
    'driver' => 's3',
    'key'    => env('AWS_ACCESS_KEY_ID'),
    'secret' => env('AWS_SECRET_ACCESS_KEY'),
    'region' => env('AWS_DEFAULT_REGION'),
    'bucket' => env('AWS_BUCKET'),
    'url'    => env('AWS_URL'),
    'endpoint' => env('AWS_ENDPOINT'),
    'use_path_style_endpoint' => env('AWS_USE_PATH_STYLE_ENDPOINT', false),
    'throw'  => false,
],
```

No changes needed ŌĆö Laravel Cloud's injected env vars map directly to these keys.

## 6.4 Student Photo Storage

Student photos are private data (PII). Never store them in a publicly accessible path. Use **signed temporary URLs** to serve them securely.

### Storing a Photo

```php
// In your controller ŌĆö store with a consistent, private path
$path = $request->file('photo')->store(
    "students/{$student->id}",
    's3'
);

// Save $path to the database (e.g. students.photo_path)
$student->update(['photo_path' => $path]);
```

### Serving a Photo

```php
// Generate a temporary signed URL (expires in 5 minutes)
$url = Storage::disk('s3')->temporaryUrl(
    $student->photo_path,
    now()->addMinutes(5)
);
```

Return the temporary URL in your API response ŌĆö the frontend uses it directly to display the photo.

### Deleting a Photo

```php
// When replacing or removing a student photo
if ($student->photo_path) {
    Storage::disk('s3')->delete($student->photo_path);
}
```

## 6.5 File Organization

Keep uploaded files organized by type under consistent prefixes:

```
sunbites-production/
  students/
    {student_id}/
      photo.jpg          ŌåÉ profile photo (one per student)
  exports/
    {branch_slug}/
      students-2026-05-26.xlsx
      wallet-report-2026-05-26.xlsx
```

> Excel exports generated by `maatwebsite/excel` should also be stored in S3 rather than served directly from memory on large datasets.

## 6.6 Local Development

For local development, keep using the `local` disk ŌĆö no S3 emulation needed. The `FILESYSTEM_DISK` env var controls which disk is active per environment.

```env
# .env (local)
FILESYSTEM_DISK=local

# Laravel Cloud staging / production
FILESYSTEM_DISK=s3
```
